## 📖 群唤醒增强插件 - 使用文档

### 简介

**群唤醒增强插件**为 AstrBot 提供精细化的群聊唤醒与拦截控制。你可以为每个 QQ 群独立配置唤醒条件、消息拦截策略、用户冷却时间、指令白名单/黑名单等，实现灵活的群管理。

### 核心能力

- ✅ **多种唤醒方式**：艾特、引用、昵称关键词、概率唤醒（支持严格纯文本模式）
- 🚫 **灵活拦截策略**：前缀拦截、关键词拦截、JSON卡片拦截、戳一戳拦截
- ⏱️ **时间控制**：用户冷却、群组/用户级持续活跃、活跃间隔
- 🔧 **指令权限控制**：按群禁用/启用特定指令或系统指令
- 🧠 **智能活跃记录**：可设置在唤醒时、LLM请求后或消息发送后记录冷却
- 🌐 **兜底规则**：为未配置自定义规则的群提供默认行为
- 🔥 **热配置**：支持 WebUI 可视化配置和指令动态修改，无需重启

---

### 安装

1. 在 `AstrBot/plugins/` 下新建目录 `群唤醒增强`
2. 将插件文件放入该目录
3. 安装依赖：`pip install pypinyin`
4. 重启 AstrBot
5. 在 WebUI 插件配置页面进行配置（或使用管理员指令）

---

### 配置方式

#### 方式一：WebUI 配置（推荐）

1. 打开 AstrBot WebUI → **插件配置** → 找到 **群唤醒增强**
2. **兜底规则**：设置全局默认规则（所有未配置自定义规则的群生效）
3. **自定义规则**：点击 **添加条目** → 选择 **自定义规则**，填写群号和其他配置项
4. 保存后立即生效

> 💡 可以为不同群组创建多条独立规则，每个群号只能归属一条规则。

#### 方式二：管理员指令热配置

通过群聊或私聊向机器人发送以下指令（需框架管理员权限），所有修改立即生效。

| 指令 | 功能 |
|------|------|
| `/所有指令` | 查看当前所有可用指令（含别名） |
| `/刷新指令` | 重新扫描并刷新指令列表（安装新插件后使用） |
| `/群规则 [群号]` | 查看指定群（或当前群）的规则配置详情 |
| `/添加群规 <群号列表> [备注]` | 为群组添加新规则（群号逗号分隔） |
| `/设置群规 <键> <值> [群号或"兜底规则"] [操作]` | 修改规则的配置项 |
| `/删除群规 <群号>` | 从规则中移除群号（若规则变空则转为占位符“无”） |
| `/删除群规 无` | 永久删除群号列表为“无”的空规则 |
| `/群规帮助` | 显示本帮助文档 |

#### `/设置群规` 用法示例

```bash
# 基本修改
/设置群规 开关 on
/设置群规 概率唤醒 0.3
/设置群规 持续活跃 60
/设置群规 活跃方式 llm请求后
/设置群规 活跃范围 用户级
/设置群规 艾特唤醒 false 兜底规则

# 列表类型（昵称唤醒、前缀拦截、禁用的指令等）
/设置群规 昵称唤醒 机器人,AI助手      # 追加（默认）
/设置群规 昵称唤醒 机器人 del        # 删除“机器人”
/设置群规 禁用的指令 help,status rep  # 替换为仅禁止 help 和 status
/设置群规 概率方式 null               # 清空概率方式列表（等同于未配置）
```

- 不指定群号时，默认操作当前群
- 指定 `兜底规则` 可修改全局默认规则
- 列表类型支持操作参数：`add`（追加，默认）、`del`（删除）、`rep`（替换）
- 使用 `null` 作为值可以清空列表

### 配置项详解

#### 基础配置

| 配置项 | 类型 | 说明 | 默认值 |
|--------|------|------|--------|
| `开关` | bool | 是否启用该规则 | `true` |
| `备注` | string | 日志中显示的规则标识 | 自动生成 |
| `群号` | list | 应用此规则的群号列表（仅自定义规则） | 空 |
| `活跃方式` | string | 何时记录活跃状态。可选值：`唤醒时`、`llm请求后`、`发送消息后` | `发送消息后` |
| `活跃范围` | string | 活跃状态的作用范围。可选值：`群聊级`、`用户级` | `群聊级` |

#### 唤醒方式

| 配置项 | 类型 | 说明 | 默认值 |
|--------|------|------|--------|
| `艾特唤醒` | bool | @机器人时唤醒 | `true` |
| `引用唤醒` | bool | 引用机器人消息且@机器人时唤醒 | `true` |
| `无艾特引用唤醒` | bool | 仅引用机器人消息（不@）时唤醒 | `false` |
| `昵称唤醒` | list | 消息包含指定关键词即唤醒，如 `["机器人","AI"]` | `[]` |
| `概率唤醒` | float | 0~1之间，每条消息以该概率随机唤醒 | `0.0` |
| `概率方式` | list | 概率唤醒适用的消息类型：`纯文本`、`卡片分享`、`图片`、`任何消息`。**当仅配置 `["纯文本"]` 时，要求消息链必须只有纯文本（无图片/卡片）** | `["纯文本"]` |

#### 拦截条件

| 配置项 | 类型 | 说明 | 默认值 |
|--------|------|------|--------|
| `前缀拦截` | list | 消息以指定前缀开头则拦截 | `[]` |
| `含有拦截` | list | 消息包含指定词则拦截 | `[]` |
| `其余拦截` | bool | 未匹配任何唤醒条件时拦截 | `false` |
| `拦截json` | bool | 拦截外部分享的JSON卡片（如快手、B站分享） | `false` |
| `拦截戳一戳` | bool | 拦截戳一戳消息 | `false` |
| `强力拦截` | bool | 开启后确保拦截后不再被其他流程唤醒 | `true` |

#### 时间控制

| 配置项 | 类型 | 说明 | 默认值 |
|--------|------|------|--------|
| `唤醒CD` | float | 单个用户唤醒后的冷却时间（秒），`-1` 表示继承兜底值 | `-1` |
| `持续活跃` | float | 唤醒后活跃状态保持的时间（秒），`-1` 表示继承兜底值 | `-1` |
| `活跃间隔` | float | 活跃期内两次唤醒的最小间隔（秒），`-1` 表示继承兜底值 | `-1` |

> 📌 时间类配置若设为 `-1`，将自动使用**兜底规则**中的对应值。

#### 指令控制

| 配置项 | 类型 | 说明 | 默认值 |
|--------|------|------|--------|
| `禁用的指令` | list | 黑名单指令，填入 `"0所有"` 则禁用所有指令 | `[]` |
| `启用的指令` | list | 白名单指令，填入 `"0所有"` 则放行所有指令 | `[]` |
| `禁用系统指令` | bool | 禁止使用系统指令（如 `/reset`、`/help` 等） | `false` |
| `禁前唤醒` | bool | 禁止错误指令唤醒LLM（如输入 `/错误指令` 时不唤醒） | `false` |

> 💡 **指令控制优先级**：`禁用的指令` > `启用的指令`。管理员不受任何指令限制。

### 高级技巧

#### 1. 让机器人在特定群聊只使用特定指令

**场景**：希望某个群只能使用 `/help` 和 `/weather` 指令，其他指令一律忽略。

**步骤**：
1. 为该群添加规则（或使用已有规则）
2. 设置 `启用的指令` 为 `["help", "weather"]`

**效果**：
- 群成员发送 `/help` → ✅ 唤醒并执行
- 群成员发送 `/weather 北京` → ✅ 唤醒并执行
- 群成员发送 `/random` → ❌ 被拦截
- 普通聊天消息（无指令前缀） → 仍按其他唤醒规则判断

> ⚠️ 一旦 `启用的指令` 非空且不含 `"0所有"`，**未在列表中的指令都将被拦截**。

#### 2. 开启兜底规则后，让特定群聊完全不使用任何规则

**场景**：全局兜底规则已开启，但想让某个群完全不受插件影响（机器人既不唤醒也不拦截）。

**步骤**：
1. 为该群添加自定义规则：`/添加群规 123456789 放行群`
2. 关闭该规则：`/设置群规 开关 false`

**效果**：
- 该群的消息**完全不被本插件处理**，直接交给框架后续流程。
- 兜底规则对该群不再生效。
- 其他未配置的群仍受兜底规则控制。

#### 3. 让普通成员完全无法唤醒机器人，但管理员可以

**步骤**：
1. 为该群添加自定义规则
2. 关闭所有唤醒方式（艾特、引用、昵称、概率）
3. 开启 `其余拦截` → `true`

**效果**：
- 普通成员的任何消息都无法唤醒机器人，被 `其余拦截` 拦截。
- 管理员不受 `其余拦截` 影响，仍可通过艾特、指令等方式唤醒。

#### 4. 用户级活跃范围：每个人独立对话冷却

**场景**：希望每个用户都能独立与机器人对话，不受群内其他人发言影响。

**配置**：`/设置群规 活跃范围 用户级`

**效果**：
- 用户 A 唤醒机器人后，其个人进入持续活跃期，活跃期内 A 的再次唤醒受 `活跃间隔` 限制。
- 用户 B 不受 A 的活跃状态影响，可以随时唤醒机器人。
- 每个用户的冷却时间（`唤醒CD`）独立计算。

### 常见问题

#### Q1：管理员是否受拦截规则影响？
**A**：管理员**不受**以下限制：用户冷却、指令禁用/启用、前缀拦截、含有拦截、其余拦截、强力拦截。但管理员仍会触发唤醒条件（艾特、昵称等），且会正常记录活跃状态。

#### Q2：`-1` 怎么用？
**A**：`唤醒CD`、`持续活跃`、`活跃间隔` 设为 `-1` 表示“继承兜底规则中对应字段的值”。便于统一管理时间参数。

#### Q3：概率唤醒的优先级为什么最低？
**A**：概率唤醒在**所有其他唤醒条件都不满足**且**未被拦截**时才会尝试，避免与显式唤醒（艾特、昵称）冲突。

#### Q4：活跃方式有什么区别？
- **唤醒时**：判定为“唤醒”后立即计时。
- **llm请求后**：等待LLM真正发出请求后才计时。
- **发送消息后**：等待机器人成功发送回复消息后才计时（最接近实际对话结束）。

#### Q5：为什么我的指令被拦截了？
**A**：检查：
1. 该群是否应用了规则（`/群规则` 查看）
2. 规则中 `禁用的指令` 是否包含该指令或 `"0所有"`
3. 规则中 `启用的指令` 是否非空且不包含该指令
4. 是否开启了 `禁用系统指令` 且该指令是系统指令
5. 指令前是否使用了正确的指令前缀（默认为 `/`）


### ✅ Q6：如何让机器人完全不处理某个群（包括不回复任何消息）？

根据需求不同，有以下几种方式：

#### 方法一：黑名单群聊（彻底拦截，管理员也受影响）
- **配置位置**：WebUI 插件配置中的 `黑名单群聊`
- **效果**：该群的所有消息在插件入口处被**直接终止**，不会进入任何唤醒/拦截判断，机器人完全忽略该群。
- **管理员**：⚠️ **也受影响**，黑名单优先级最高，不检查管理员身份。
- **适用场景**：想让机器人完全无视某个群（包括管理员）。

#### 方法二：自定义规则 + 关闭所有唤醒 + 开启其余拦截（普通成员全拦截，管理员可唤醒）
- **配置步骤**：
  1. 为该群添加自定义规则（`/添加群规 <群号>`）
  2. 关闭所有唤醒方式：
     - `艾特唤醒` → `false`
     - `引用唤醒` → `false`
     - `无艾特引用唤醒` → `false`
     - `昵称唤醒` → 清空 `[]`
     - `概率唤醒` → `0`
  3. 开启 `其余拦截` → `true`
- **效果**：
  - **普通成员**：任何消息都无法唤醒机器人，被 `其余拦截` 拦截。
  - **管理员**：`其余拦截` 对管理员无效，管理员仍可通过艾特、指令等方式正常唤醒。
- **注意**：如果希望管理员也无法唤醒，请使用方法一（黑名单）。

#### 方法三：自定义规则 + 开关设为 `false`（完全放行，不处理）
- **配置**：`/设置群规 开关 false`
- **效果**：该群的消息**完全不被本插件处理**，直接交给框架后续流程。机器人是否回复取决于其他插件或框架默认行为。
- **注意**：这种方式不会拦截任何消息，只是“不参与本插件的规则判断”。

---

**总结对比**：

| 方式 | 普通成员 | 管理员 | 消息是否被本插件处理 |
|------|----------|--------|----------------------|
| 黑名单 | ❌ 拦截 | ❌ 拦截 | 入口直接终止 |
| 关闭唤醒 + 其余拦截 | ❌ 拦截 | ✅ 可唤醒 | 进入规则判断后拦截 |
| 开关 = false | ✅ 放行 | ✅ 放行 | 完全不处理 |

根据你的实际需求选择最合适的方式即可。

#### Q7：修改配置后需要重启吗？
**A**：不需要。无论是 WebUI 保存还是指令修改，所有变更都**立即生效**。
