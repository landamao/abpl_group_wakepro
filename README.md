# 群唤醒增强插件使用文档（v8.4.0）

## 简介

群唤醒增强插件允许您为每个QQ群单独配置机器人的唤醒条件和消息拦截规则，实现精细化的群聊管理。支持：

- **多种唤醒方式**：艾特、引用、昵称关键词、概率唤醒
- **灵活拦截策略**：前缀拦截、关键词拦截、JSON卡片拦截、戳一戳拦截
- **时间控制**：用户冷却、群组持续活跃、活跃间隔
- **指令权限控制**：按群禁用/启用特定指令或系统指令
- **智能活跃记录**：可设置在唤醒时、LLM请求后或消息发送后记录冷却（v8.4.0新增）
- **兜底规则**：为未配置的群提供默认行为

插件内置WebUI可视化配置界面，无需手动编辑JSON文件。

---

## 安装

1. 将插件文件夹放入 `AstrBot/plugins/` 目录
2. 执行 `pip install pypinyin` 安装依赖（若已安装可跳过）
3. 重启AstrBot或执行热重载
4. 在WebUI中配置规则（详见下方说明）

---

## 配置说明

### 通过WebUI配置（推荐）

1. 打开AstrBot WebUI，进入 **插件配置** 页面
2. 找到 **群唤醒增强** 插件，点击 **添加条目** → 选择 **自定义规则**
3. 填写规则配置表单（各配置项详见下表）
4. 保存配置，规则立即生效

> 💡 可以为不同群组创建多条独立规则，群号不可重复。

### 兜底规则配置

在插件配置页面中，**兜底规则**部分用于设置所有未配置自定义规则的群的默认行为。兜底规则开关决定是否启用。

---

## 配置项详解

### 基础配置

| 配置项 | 类型 | 说明 | 默认值 |
|--------|------|------|--------|
| `开关` | bool | 是否启用该规则 | `true` |
| `备注` | string | 日志中显示的规则标识 | 自动生成 |
| `群号` | list | 应用此规则的群号列表（仅自定义规则） | 空 |
| `活跃方式` | string | 何时记录群活跃和用户冷却。可选值：`唤醒时`、`llm请求后`、`发送消息后` | `唤醒时` |

### 唤醒方式

| 配置项 | 类型 | 说明 | 默认值 |
|--------|------|------|--------|
| `艾特唤醒` | bool | @机器人时唤醒 | `true` |
| `引用唤醒` | bool | 引用机器人消息且@机器人时唤醒 | `true` |
| `无艾特引用唤醒` | bool | 仅引用机器人消息（不@）时唤醒 | `false` |
| `昵称唤醒` | list | 消息包含指定关键词即唤醒，如 `["机器人","AI"]` | `[]` |
| `概率唤醒` | float | 0~1之间，每条消息以该概率随机唤醒 | `0.0` |

### 拦截条件

| 配置项 | 类型 | 说明 | 默认值 |
|--------|------|------|--------|
| `前缀拦截` | list | 消息以指定前缀开头则拦截，如 `["【广告】","【警告】"]` | `[]` |
| `含有拦截` | list | 消息包含指定字符串则拦截 | `[]` |
| `拦截json` | bool | 拦截外部分享的JSON卡片（如快手、B站分享） | `false` |
| `拦截戳一戳` | bool | 拦截戳一戳（拍一拍）事件 | `false` |
| `其余拦截` | bool | 未匹配任何唤醒条件时，是否拦截该消息 | `false` |
| `强力拦截` | bool | 开启后确保拦截生效（防止后续插件唤醒） | `false` |

### 时间控制

| 配置项 | 类型 | 说明 | 默认值 |
|--------|------|------|--------|
| `唤醒CD` | float（秒） | 用户唤醒一次后的冷却时间（管理员不受影响）。设为 `-1` 则继承兜底规则 | `-1` |
| `持续活跃` | float（秒） | 唤醒后群组保持活跃的时间，活跃期内可响应后续消息。`-1` 继承兜底 | `-1` |
| `活跃间隔` | float（秒） | 活跃期内，两次唤醒的最小间隔（艾特/引用不受限）。`-1` 继承兜底 | `-1` |

### 指令控制

| 配置项 | 类型 | 说明 | 默认值 |
|--------|------|------|--------|
| `禁用的指令` | list | 禁止使用的指令名（不加前缀）。填 `["0所有"]` 禁用所有指令 | `[]` |
| `启用的指令` | list | 仅允许使用的指令名，优先级低于禁用列表。填 `["0所有"]` 放行所有指令 | `[]` |
| `禁用系统指令` | bool | 禁止使用系统管理指令（如 `/reset`、`/op`、`/help` 等） | `false` |
| `禁前唤醒` | bool | 禁止通过错误指令唤醒LLM（即输入不存在的前缀指令时，不唤醒机器人） | `false` |

> **优先级说明**：`禁用的指令` > `启用的指令`。管理员不受任何指令限制。

## 活跃方式详解（v8.4.0）

- **唤醒时**（默认）  
  只要用户通过艾特、引用、昵称关键词或概率成功触发唤醒（即进入 `唤醒()` 方法），就立即记录群活跃时间和用户冷却。**适合希望严格限制触发次数的场景**。

- **llm请求后**  
  仅在机器人实际调用LLM并返回响应后，才记录活跃和冷却。如果消息因其他原因（如被其他插件拦截、LLM返回错误）未产生回复，则不消耗冷却。**适合避免因误触发浪费冷却的精细控制场景**。

- **发送消息后**  
  在机器人完整发送完回复消息后才记录（支持分段回复插件，确保所有消息段发送完毕）。**最接近用户感知的“机器人已回应”时机**。
  - 实测框架只有在全部消息发送完成后才触发，不知道不同版本是否有变化，不过不会重复触发

## 管理员指令

以下指令需要**群主/管理员**或在AstrBot配置的 `admins_id` 中。

| 指令 | 格式 | 说明 |
|------|------|------|
| `/所有指令` | `/所有指令` | 查看当前所有可用的机器人指令（含别名） |
| `/刷新指令` | `/刷新指令` | 重新扫描并刷新指令列表（安装新插件后使用） |
| `/群规则` | `/群规则 [群号]` | 查看指定群（或当前群）的规则配置详情 |
| `/添加群规` | `/添加群规 <群号列表> [备注]`<br>示例：`/添加群规 123456,789012 测试群` | 为群组添加新规则（群号用英文逗号分隔） |
| `/设置群规` | `/设置群规 <键> <值> [群号或"兜底规则"] [操作]`<br>详见下方示例 | 修改指定规则的配置项 |
| `/群规帮助` | `/群规帮助` | 显示本帮助文档 |

### `/设置群规` 详细用法

- **基本修改**  
  `/设置群规 开关 on`  
  `/设置群规 概率唤醒 0.3`  
  `/设置群规 持续活跃 60`  
  `/设置群规 活跃方式 llm请求后`  
  `/设置群规 艾特唤醒 false 兜底规则`

- **列表类型（昵称唤醒、前缀拦截、禁用的指令等）支持操作参数**  
  - `add`：追加（默认）  
  - `del`：删除  
  - `rep`：替换  

  示例：  
  `/设置群规 昵称唤醒 机器人,AI助手`（追加机器人、AI助手）  
  `/设置群规 昵称唤醒 机器人 del`（删除机器人）  
  `/设置群规 禁用的指令 help,status rep`（替换为仅禁止help和status）

- **特殊值**  
  - `0所有`：在 `禁用的指令` 或 `启用的指令` 中使用，表示全部指令  
  - `-1`：在时间类配置中使用，表示继承兜底规则的对应值

> 💡 不指定群号时，默认操作当前群；指定 `兜底规则` 可修改全局默认规则。

## 注意事项

1. **管理员特权**：群主、群管理员以及在 `admins_id` 中配置的用户，不受拦截规则和指令限制，但依然受黑名单插件影响。
2. **配置优先级**：自定义规则 > 兜底规则。若群号匹配了自定义规则，则兜底规则对该群无效（除非自定义规则关闭了总开关）。
3. **指令前缀**：插件使用AstrBot全局配置的 `wake_prefix`（默认为 `/`），示例中均以 `/` 演示，请根据实际情况调整。
4. **刷新指令**：当安装或卸载了其他插件（新增指令）后，建议执行 `/刷新指令` 以更新指令列表。
5. **JSON拦截**：仅拦截消息段类型为 `Json` 的消息（通常为外部分享卡片），普通文本不受影响。
6. **兜底规则开关**：若关闭兜底规则，未配置自定义规则的群将**完全不受本插件管理**（即所有消息默认放行，不拦截也不强制唤醒）。
7. **活跃方式切换**：修改 `活跃方式` 后，原有内存中的活跃状态不会立即清除，但新规则会立即生效。

## 常见问题

**Q：为什么设置了昵称唤醒但没反应？**  
A：检查昵称唤醒列表中的关键词是否与消息完全匹配（支持子串匹配）。例如设置 `["机器人"]`，消息含“机器人”即可唤醒。

**Q：用户冷却期间，艾特机器人还能唤醒吗？**  
A：冷却期间该用户的所有消息都会被拦截，包括艾特和引用。如果希望艾特忽略冷却，可将 `唤醒方式` 设置为 `llm请求后` 或 `发送消息后`，并在规则中关闭 `禁前唤醒`（但冷却仍会生效）。

**Q：如何让某个群完全不响应任何消息？**  
A：创建自定义规则，将 `开关` 设为 `false`，或将 `其余拦截` 设为 `true` 且关闭所有唤醒方式。

**Q：修改配置后需要重启吗？**  
A：不需要，所有配置修改立即生效。

**Q：可以同时使用多个规则吗？**  
A：每个群只能匹配一个自定义规则（按群号匹配）。如果多个规则包含同一群号，后添加的会覆盖前一个（日志会警告）。

**Q：活跃方式中的“llm请求后”和“发送消息后”有什么区别？**  
A：`llm请求后` 在 LLM 生成回复后立即记录（但消息可能尚未发送到聊天软件）；`发送消息后` 等待消息实际发送完成（对分段回复更准确）。绝大多数情况选 `llm请求后` 即可。
